# 开发规则

- 修改现有代码时，**必须保留所有原有注释**
- 新增代码时，**必须添加必要的注释**
- 原有代码的注释原则上**不能被修改**，除非是必要的修改，比如功能已经发生变化，注释已经过时了
- 原有代码的转义符号**不能被修改**，除非是完全不需要转义的情况
- 项目使用vokex作为应用框架，vokex 使用npm link链接到项目中，项目中直接引入vokex即可使用。不要在项目中安装vokex的依赖。
- vokex的文档参考：[vokex文档](https://raw.githubusercontent.com/zstings/vokex/refs/heads/master/api-doc.md)
